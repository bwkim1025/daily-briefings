# 브리핑 편집 원칙 (Daily Briefings)

> 모든 자동 브리핑(파이낸셜·인터내셔널·메디컬·건강)에 공통 적용

## 핵심 원칙: 당일 뉴스 우선

### 1. 시간 우선순위
- **1순위 (필수):** 24시간 이내 발생·발표·공시된 이벤트
- **2순위 (조건부):** 전일 마감 지표·종가 (예: KST 기준 미국 증시는 항상 전일 데이터)
- **3순위 (제한적):** 24시간 이상 지난 사건이라도 "오늘 시장이 새롭게 반응"하거나 "오늘 처음으로 가격·정책에 반영"되는 경우에만

### 2. 비율 가이드
- 카테고리당 3개 헤드라인 중 **최소 2개는 24시간 이내 신선한 이벤트**여야 함
- 나머지 1개는 다중일 진행 이벤트(예: 정상회담 일정, 협상 진척)도 가능

### 3. 반복 금지
- 어제 이미 다룬 동일 토픽은 **새로운 진전**(추가 보도·정책 반응·가격 변동·발언 추가 등)이 있을 때만 재등장
- 단순 재언급("어제 발표된 X")은 제외

### 4. 시제 명확화
- 본문에 정확한 일자 명시 (예: "5월 7일 오전", "5월 6일 종가", "직전 분기")
- 모호한 표현 지양 ("최근", "한동안", "근래" 등)

### 5. 카테고리별 적용 강도
- **파이낸셜·인터내셔널 (매일):** 위 원칙 엄격 적용
- **메디컬·건강 (주간):** "이번 주" 기준 유지, 24h 우선순위는 완화 가능
  (논문 publication·정책 발표는 주 단위가 자연스러움)

### 6. 메디컬·건강 — 펼친 카드 정보량 (필수)
헬스/메디컬 페이지의 아코디언을 펼쳤을 때 **논문 내용을 이해할 수 있는 수준**까지 정보를 제공해야 함.
- `#### 어떤 연구` (또는 `#### Why`/`#### 선정 이유`) — 연구 배경·중요성 (1~2문단)
- `#### 초록` (필수, 신규) — 연구 설계·주요 측정치·핵심 결과를 3~5문장. 원논문 abstract를 한국어로 풀어쓰거나 abstract 일부를 그대로 인용해도 좋음.
- `#### 주요 결과` (선택, 권장) — 대표 그림·표·핵심 수치를 글머리표 또는 markdown 표로 압축. 예: "1차 결과: HbA1c −0.7%p (p<0.001)".
- `#### 우리 가족에 적용` (또는 `#### 임상 적용`) — 행동·임상 함의
- `#### 주의할 점` — 한계·신중하게 해석할 부분
- `#### Source` — 원문 URL. 형식: `- 매체/저널 [https://원본주소]` 또는 `- [표시명](https://원본주소)`. 모든 링크는 렌더링 시 자동으로 클릭 가능하게 변환됨.

> **분량 가이드:** 2026-05-08 health 브리핑(`health/briefings/2026-05-08.md`)을 **참조 분량**으로 삼는다. 항목당 초록 250~320자, 주요 결과는 글머리표 3~5개 또는 4~5행 표 정도. 이 수준이면 논문을 이해하는 데 충분 — 더 늘리지 말 것. 접힌 상태(헤드라인+한 줄 요약)는 기존 분량 유지.

### 7. 작성 모델 표기 (필수, 모든 브리핑 공통)
파일 끝에 `## AUTHOR` 섹션을 반드시 포함. 어느 Claude 모델이 작성했는지 사용자가 한눈에 확인할 수 있어야 함.

**형식 (한 줄):**
```
## AUTHOR
Claude Opus 4.7 · claude-opus-4-7 · 2026-05-08 07:00 KST
```

- 첫 토큰: 사람이 읽기 쉬운 모델명 (예: `Claude Opus 4.7`, `Claude Sonnet 4.6`)
- 둘째 토큰: API에서 사용된 정확한 모델 ID (예: `claude-opus-4-7`, `claude-sonnet-4-6`, `claude-haiku-4-5-20251001`)
- 셋째 토큰: 작성 시각 (KST, `YYYY-MM-DD HH:MM KST`)

**자기 식별:** 스케줄드 태스크는 자신이 실행 중인 모델 ID를 시스템 컨텍스트(env)에서 읽어 적는다. 모델 ID를 임의로 만들지 말 것. 확실하지 않으면 사람이 읽는 이름만 적고 ID 자리는 `(미확인)`로 표시.

**렌더링:** 4개 페이지(financial/international/medical/health) 모두 본문 끝에 작은 글씨로 자동 표시됨. `## AUTHOR` 섹션이 없으면 "작성 모델 미기록"으로 표시되어 누락이 눈에 띄도록 함.

## 좋은 예시 vs 나쁜 예시 (파이낸셜)

❌ **나쁜 예:** "Apple 2분기 매출 1,112억 달러 — 4월 30일 발표"
   (이미 며칠 지남)

✅ **좋은 예:** "Apple, 5월 6일 인텔 파운드리 협의 보도에 +2.6% — 1분기 실적 견인"
   (오늘 새 보도 + 어제 발표된 실적과 연결)

❌ **나쁜 예:** "Fed 4월 28일 동결, 시장 영향 분석"
   (열흘 가까이 지난 이벤트 단순 재언급)

✅ **좋은 예:** "5월 7일 발표 4월 PCE 0.3% MoM — Fed 6월 동결 가능성 70%로 상승"
   (오늘 발표된 지표 + 시장 반응)
